Commands — detailed command reference
Use an explicit command. Bare `tmux-a2a-postman` prints usage; it does not start the daemon.

Default operator surface

send-heredoc
  Compose and deliver a message atomically with an explicit quoted heredoc.
  Output: JSON
  Usage:
    tmux-a2a-postman send-heredoc --to <node> <<'POSTMAN_BODY'
    literal $(not executed), `backticks`, and $HOME
    POSTMAN_BODY
  Flags:
    --to <node>          Recipient node name (required)
    --no-reply           Mark this message as not requiring a reply
    --reply-required     Mark this message as requiring a reply
    --reply-to <message> Message id for fallback closure or traceability
    --fills-input-request-id <id>
                         Exact input request id this message fills

send
  Body argv is disabled; use send-heredoc for message delivery.
  This command returns explicit safety guidance for shell-expansion risks.

pop
  Read and archive the oldest unread inbox message.
  Output: JSON
  Empty inbox: {"status":"empty"}
  Node is auto-detected from tmux pane title.

capture-profile
  Capture one explicit Go runtime profile from the running daemon.
  Profiling has no default listener or background collector.
  Output: JSON for --output PATH; raw profile bytes for --output -
  --output PATH refuses to overwrite an existing file; use --force to allow it.
  Usage:
    tmux-a2a-postman capture-profile --type heap --output ./postman-heap.pprof
    tmux-a2a-postman capture-profile --type goroutine --output -
  Flags:
    --type heap|goroutine   Profile type to capture (required)
    --output -|PATH         Explicit destination (required)
    --max-bytes N           Maximum profile bytes (default: 16777216)
    --force                 Overwrite an existing output file (default: refuse)

get-status
  Print canonical session status JSON for agents and scripts.
  Uses the current tmux session.

get-status-oneline
  Print compact all-session status for quick agent coordination.
  Shape: [0]🔷🟡:🟢 [1]🔴
  Window groups are colon-separated emoji runs with no literal window labels.

inspect-input
  Inspect open reply-required work by message_id or input_request_id.
  Output: JSON
  Usage:
    tmux-a2a-postman inspect-input --id <message_id-or-input_request_id>

inspect-daemon-submit
  Inspect daemon-submit timeout state by request_id.
  Output: JSON
  Usage:
    tmux-a2a-postman inspect-daemon-submit --id <request_id>

inspect-message
  Inspect persisted message metadata or content by message_id.
  Output: JSON by default; --path and --body print focused text.
  Usage:
    tmux-a2a-postman inspect-message --id <message_id>
    tmux-a2a-postman inspect-message --id <message_id> --path
    tmux-a2a-postman inspect-message --id <message_id> --body

execute-bash
  Run a bash command through command approval choreography.
  Output: command stdout/stderr; refusal metadata on stderr.
  Usage:
    tmux-a2a-postman execute-bash --label <label> --command <bash>

inspect-command-approvals
  Inspect pending and decided command approval threads.
  Output: JSON

version
  Print the build version JSON.
  Usage:
    tmux-a2a-postman version
    tmux-a2a-postman --version
  Output: JSON

Lifecycle and recovery

start
  Start the tmux-a2a-postman daemon.

stop
  Stop the running daemon for the current tmux session.
  Output: JSON
  Sends SIGTERM and waits up to 10 seconds for exit.
  Exits 0 if no daemon is running (idempotent).

help [topic]
  Show help overview or detailed topic page.
  Topics: messaging, directories, config, commands, start, stop, send-heredoc, send, pop, capture-profile, get-status, get-status-oneline, inspect-input, inspect-daemon-submit, inspect-message, execute-bash, inspect-command-approvals, version, help
