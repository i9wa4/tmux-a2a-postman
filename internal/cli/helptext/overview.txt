tmux-a2a-postman — A2A message routing daemon for tmux panes

AI agents send and receive messages. Each message is routed to the recipient
by the daemon. Use send-heredoc to send and pop to claim/open.
Use an explicit command. Bare `tmux-a2a-postman` prints usage; it does not start the daemon.

Quick Start:
  1. Start daemon:    tmux-a2a-postman start
  2. Send a message:
     tmux-a2a-postman send-heredoc --to <node> <<'POSTMAN_BODY'
     message text
     POSTMAN_BODY
  3. Claim/open msg:  tmux-a2a-postman pop

Key Concepts:
  Node       An AI agent identified by its tmux pane title.
  Edge       A bidirectional routing rule between two nodes (configured in edges).
  Envelope   YAML frontmatter at the top of each message file:
               ---
               params:
                 from: <sender>
                 to: <recipient>
                 timestamp: <ISO 8601>
               ---

Default operator surface:
  send-heredoc               Send a message with an explicit quoted heredoc
  pop                        Claim and archive the oldest unread inbox message
  get-status                 Print canonical session status JSON
  get-status-oneline         Print compact all-session status
  inspect-input              Inspect open reply-required work by id
  inspect-daemon-submit      Inspect daemon-submit timeout state by id
  inspect-message            Inspect persisted message content by id
  execute-bash               Run bash through command approval choreography
  inspect-command-approvals  Inspect command approval threads
  version                    Print the build version JSON

Lifecycle and recovery:
  start                      Start the daemon (single-column TUI)
  stop                       Stop the running daemon for this tmux session
  Use `help commands` for the full command list and diagnostic topics.

Messaging Protocol:
  send-heredoc --to <node> <<'POSTMAN_BODY' Send a message from quoted heredoc stdin
  pop                                        Claim and archive oldest message
  get-status                                Inspect agent-readable session status
  get-status-oneline                        Inspect compact all-session status
  inspect-input --id <id>                   Inspect an open input request by id
  inspect-daemon-submit --id <request_id>   Inspect daemon-submit timeout state by id
  inspect-message --id <message_id>         Inspect persisted message content by id
  execute-bash --label <label> --command <bash>
                                           Run bash through approval choreography
  inspect-command-approvals                 Inspect command approval threads
  version                                   Print the build version JSON
