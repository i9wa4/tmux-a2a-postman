Messaging — send and receive messages

Quick workflow (agents):
  Send:
    tmux-a2a-postman send-heredoc --to <node> <<'POSTMAN_BODY'
    message text
    POSTMAN_BODY
  Read:  tmux-a2a-postman pop

Safe body input:
  Use send-heredoc with a quoted heredoc delimiter for frequent agent-to-agent
  messages. The quoted delimiter preserves command substitutions, backticks,
  $HOME variables, quotes, code fences, and shell examples as literal text.
  Do not pass message text through argv, file-body shortcuts, or generic
  pipe-oriented guidance.

Reply workflow:
  tmux-a2a-postman send-heredoc --to <sender> <<'POSTMAN_BODY'
  DONE: ...
  POSTMAN_BODY

Reply-required messages carry an input_request_id. Exact replies use:
  tmux-a2a-postman send-heredoc --to <sender> --fills-input-request-id <input_request_id> --reply-to <message_id> <<'POSTMAN_BODY'
  DONE: ...
  POSTMAN_BODY

The exact input request id clears the required action; --reply-to is retained for
traceability and fallback message-link closure. Replies stay on the live
session daemon automatically.

Inspect open reply-required work without popping inbox mail:
  tmux-a2a-postman inspect-input --id <message_id-or-input_request_id>

get-status also exposes open input-request detail arrays under
nodes[*].flow.input_requests.input_required and waiting_on_input.

Envelope format (YAML frontmatter):
  ---
  params:
    from: <sender node name>
    to: <recipient node name>
    messageId: <message filename>
    replyPolicy: <none|required>
    input_request_id: <required reply token>
    timestamp: <ISO 8601 timestamp>
  ---

Sender is auto-detected from the tmux pane title (no --from flag).
`send-heredoc` and `pop` print JSON by default.
