send — legacy command name

Message body sending is disabled on `send`.

Use the heredoc-explicit command instead:

  tmux-a2a-postman send-heredoc --to <node> <<'POSTMAN_BODY'
  literal $(not executed), `backticks`, and $HOME
  POSTMAN_BODY

Reason:
  AI-generated message text often contains shell snippets, backticks,
  command substitutions, variables, quotes, or code fences. Passing that body
  through argv, file shortcuts, or generic pipe-oriented guidance can cause
  shell-expansion mistakes. The supported fast path is an explicit quoted
  heredoc using <<'POSTMAN_BODY'.
