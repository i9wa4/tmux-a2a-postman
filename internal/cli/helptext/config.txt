Config — core runtime model

Primary files:
  $XDG_CONFIG_HOME/tmux-a2a-postman/postman.toml
  $XDG_CONFIG_HOME/tmux-a2a-postman/postman.md
  (fall back to ~/.config/tmux-a2a-postman/...)

postman.toml is optional. Embedded defaults from postman.default.toml are used
when no user TOML exists. A minimal postman.md can contain only Mermaid edges;
nodes referenced by those edges are materialized automatically. Mark the
human-facing startup PING target with the Mermaid ui_node class.

Runtime state model:
  Core visible states: ready, waiting, pending, stale
  Session fallback: unavailable (this daemon does not own canonical health)
  get-health, get-health-oneline, and the default TUI read the same canonical health contract.

Quick reading guide:
  visible_state in get-health JSON answers what the node looks like now
  pane hints answer that delivery reached a recipient inbox

Core config:
  edges                            Bidirectional routes between nodes
  ui_node                          Optional target filter for startup auto-PING; prefer Mermaid class <node> ui_node
  auto_enable_new_sessions         Auto-enable sessions with configured node panes (default: true)
  message_footer                   Footer appended to stored send mail
  skill_path                       postman.md YAML frontmatter skill catalog sources
  session_scan_interval_seconds    Lightweight tmux session-list refresh interval (default: 0.25)
  auto_ping_delay_seconds          Delay before first auto-PING for newly appeared/replacement nodes (default: 20; 0 = immediate)
  notification_template            Pane hint rendered when mail arrives
  min_delivery_gap_seconds         Same-route delivery gap for duplicate control
  retention_period_days            Inactive runtime cleanup window (default: 50; 0 = disabled)
  pane_capture_tail_lines          Recent lines scanned for compaction markers (default: 100; 0 = visible pane only)

Edge syntax:
  edges = [
    "node-a --- node-b",   # bidirectional: a<->b
    "node-b --- node-c",   # bidirectional: b<->c
  ]
